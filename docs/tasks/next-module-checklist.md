### 1. 题库模块

- **知识分类表 `knowledge_categories`**
  - `category_id`：主键
  - `parent_id`：父级分类 ID
  - `category_code`：分类编码，树形层级编码，子级编码基于父级编码追加固定长度片段生成
  - `category_name`：分类名称
  - `order_num`：显示排序
  - `ancestors`：祖级路径，便于树查询，格式可参考部门管理
  - `status`：状态
  - `remark`：备注
  - `creator_id`：创建人
  - `is_deleted`：软删除标记

> 编码规则建议：根节点为基础编码，每新增一级就在父级 `category_code` 后追加一段固定长度编码，整体效果类似行政区划编码或部门层级编码。

- **题目表 `questions`**
  - `question_id`：主键
  - `knowledge_category_id`：关联知识分类
  - `type`：题型，支持单选、多选、判断、主观题
  - `auto_grading`：是否自动判分
  - `tags`：JSON，扩展标签
  - `content`：题干富文本
  - `attachments`：JSON，关联的独立附件列表
  - `options`：JSON，选项内容
  - `standard_answer`：JSON，标准答案，需兼容客观题判分和主观题自动判分
  - `partial_score_rule`：JSON，多选题部分得分规则，至少支持“漏选且全对得半分”和“漏选得0分”
  - `analysis`：解析说明
  - `difficulty`：难度等级
  - `creator_id`：创建人
  - `is_deleted`：软删除标记

> `standard_answer` 建议按结构化 JSON 保存，避免后续自动判分难以扩展：
>
> - 客观题：`{ "type": "objective", "answers": ["A", "C"] }`
> - 主观题：`{ "type": "subjective", "keywords": ["牛顿第二定律", "F=ma"], "match_mode": "any", "reference_answer": "..." }`
>
> `partial_score_rule` 建议至少支持：
>
> - `{"mode":"half_if_partial_correct"}`
> - `{"mode":"zero_if_not_full_correct"}`

### 2. 附件媒体模块

- **附件表 `attachments`**
  - `attachment_id`：主键
  - `file_url`：文件访问地址或相对路径
  - `file_type`：文件类型，如图片、音频
  - `bucket_name`：存储桶名称
  - `object_name`：对象存储中的对象名
  - `uploader_id`：上传人
  - `is_deleted`：软删除标记

### 3. 试卷模块

- **试卷模板表 `exam_papers`**
  - `paper_id`：主键
  - `title`：试卷标题
  - `total_score`：总分
  - `pass_score`：及格分
  - `paper_structure`：JSON，试卷结构，支持固定题与随机题混合配置
  - `creator_id`：创建人
  - `is_deleted`：软删除标记

> `paper_structure` 建议支持两类结构项：
> - 固定题：`{ "type": "fixed", "question_id": 1, "score": 2 }`
> - 随机题：`{ "type": "random", "filter": { "knowledge_category_id": 1001, "type": "single", "difficulty": "medium" }, "count": 5, "score_per_item": 2, "generate_timing": "on_exam" }`

### 4. 考试与分发模块

- **考试场次表 `exams`**
  - `exam_id`：主键
  - `paper_id`：关联试卷模板
  - `title`：考场名称
  - `time_mode`：时间模式，`FIXED` / `DURATION_ONLY` / `UNLIMITED`
  - `start_time`：固定起止模式下的开始时间
  - `end_time`：固定起止模式下的结束时间
  - `duration_mins`：答题时长，固定起止和仅限时长模式有效
  - `status`：状态，如草稿、发布、撤回、归档
  - `is_deleted`：软删除标记

- **考试分发目标表 `exam_targets`**
  - `target_id`：主键
  - `exam_id`：关联考试
  - `target_type`：目标类型，`CLASS` 或 `USER`
  - `target_value`：对应班级 ID 或用户 ID
  - `is_deleted`：软删除标记

### 5. 试卷快照模块

- **学生专属试卷快照表 `exam_paper_snapshots`**
  - `snapshot_id`：主键
  - `exam_id`：关联考试
  - `user_id`：考生 ID
  - `paper_json`：JSON，当前考生本次考试的完整试卷快照，不含标准答案
  - `standard_answer_json`：JSON，当前快照对应的标准答案与评分规则
  - `is_deleted`：软删除标记

> 约束建议：`exam_id + user_id` 唯一，确保单次作答场景下每名考生只有一份正式快照。

### 6. 答卷与成绩模块

- **答卷记录表 `exam_records`**
  - `record_id`：主键
  - `exam_id`：关联考试
  - `user_id`：考生 ID
  - `snapshot_id`：关联试卷快照
  - `start_time`：真实开卷时间
  - `user_answers`：JSON，考生作答内容，支持服务端暂存覆写
  - `submit_time`：最终交卷时间
  - `status`：状态，如答题中、待阅卷、已阅卷
  - `total_score`：最终得分
  - `is_deleted`：软删除标记

- **答卷明细表 `exam_record_details`**
  - `detail_id`：主键
  - `record_id`：关联答卷
  - `question_id`：关联原题 ID
  - `user_answer`：单题作答内容
  - `score`：单题得分
  - `is_correct`：是否正确
  - `is_deleted`：软删除标记

> 约束建议：`exam_records` 也按 `exam_id + user_id` 唯一，V1 默认每位考生每场考试只有一条正式答卷记录。

### 7. 学生端试卷中心

- **学生试卷中心视图**
  - 展示学生可见的全部试卷/考试
  - 已完成试卷显示分数、状态、交卷时间
  - 未开始或可作答试卷显示“开始考试”入口
  - 支持查看历史试卷详情和历史答题记录

> 交互要求：学生点击“开始考试”后，使用新标签页打开沉浸式答题页面。
